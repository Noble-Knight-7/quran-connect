## 🌙 The Problem

Millions of Muslims reconnect with the Quran during Ramadan — but maintaining that connection afterwards is hard. Life gets busy. Without structure, habit, and community, even the most sincere intentions fade.

**Quran Connect** is built to solve this. It's not just a Quran reader. It's a personal companion that tracks your journey, builds your habits, challenges your community, and uses AI to guide your reflection — every single day.

---

## ✨ Features

### 📖 Quran Reader

A full, beautifully rendered Quran reader powered by the **Quran Foundation Content API**, featuring:

- All 114 Surahs with Arabic text (Uthmani script)
- English translations and word-by-word transliteration
- Direct navigation by Surah — with smooth scroll-to-verse support

### 🔗 Quran Foundation Account Sync

Users can connect their **Quran Foundation account** via OAuth 2.0. Once connected:

- The app reads your **latest reading session** and surfaces a "Continue Reading" card right on the home screen
- Your reading **streak and activity** are pulled directly from your QF account
- Bookmarks and activity goals sync through the **QF User API**

### 🤖 AI Daily Reconnect Plan (Powered by Gemini)

Every day, a unique reconnect plan is generated by **Google Gemini AI** on the backend:

- A curated verse tailored to the spiritual season
- A focused action step (e.g., _"Read Ayat al-Kursi and sit with its meaning for one minute"_)
- A deep reflection prompt to journal on
- Users can mark the plan complete and track their consistency over time

### 🧠 Daily Quiz

A knowledge-based Quran quiz that tests your familiarity with the Book:

- Drawn from a rich local question bank covering Surah names, verse meanings, and Islamic knowledge
- One quiz per day — results stored to Firebase and reflected in your profile

### 🔥 Streak Tracker

A persistent streak system built on **Firebase Firestore**:

- Tracks consecutive days of Quran activity
- Displays current streak, longest streak, total reading days, and unique Surahs read
- Protects against timezone edge cases using a custom date utility

### 🏅 Badges & Achievements

An unlockable badge system that celebrates milestones:
| Badge | Requirement |
|---|---|
| 🌱 First Step | Log your first reading day |
| 🔥 Week Warrior | Reach a 7-day streak |
| ⚡ Monthly Master | Reach a 30-day streak |
| 📚 Dedicated Reader | Read on 10 different days |
| 💎 Centurion | Read on 100 different days |
| 👑 Khatam | Complete your first full Quran |

### 🌍 Community & Leaderboard

A live community page where users can:

- See a **global leaderboard** ranked by streak and total reading days
- Participate in **monthly community challenges** (e.g., _Juz Amma Quest_, _The Protector — 7 nights of Surah Al-Mulk_)
- Track their position relative to others and stay motivated

### ✨ Verse of the Day

A randomly selected verse from the entire Quran is shown on the home screen, refreshed every 6 hours, with:

- Full Arabic text in Uthmani script
- Word-by-word transliteration
- A direct link to read the full Surah or reflect on the verse

## 🔌 API Integrations

This project fulfills the hackathon's requirement of using **at least one Content API and at least one User API** from the Quran Foundation:

### Content APIs Used

| API                          | Usage                                                     |
| ---------------------------- | --------------------------------------------------------- |
| `GET /verses/by_key/:key`    | Fetch Verse of the Day with Arabic text + transliteration |
| `GET /chapters`              | Load the full list of 114 Surahs                          |
| `GET /chapters/:id`          | Fetch Surah metadata (name, verse count)                  |
| `GET /verses/by_chapter/:id` | Load all verses for the Surah reader                      |

### User APIs Used

| API                          | Usage                                                                |
| ---------------------------- | -------------------------------------------------------------------- |
| **OAuth 2.0 Token Exchange** | Connect a user's Quran Foundation account                            |
| **Reading Sessions**         | Pull the user's latest reading position to enable "Continue Reading" |
| **Streak Data**              | Display reading streak from the user's QF account                    |
| **Activity & Goals**         | Reflect QF-tracked activity in the dashboard                         |

## 🤖 AI Integration

The **Daily Reconnect Plan** is generated by **Google Gemini** (`gemini-1.5-flash`) via a custom Express.js backend. The server:

1. Selects a seed verse reference from a 30-day curated plan
2. Fetches the full verse text from the Quran Foundation API
3. Sends a structured prompt to Gemini to generate a spiritually resonant action step and reflection question
4. Caches the result per user per day in Firestore to avoid redundant API calls

## 🚀 Getting Started

### Prerequisites

- Node.js v18+
- A Firebase project (Firestore + Google Auth enabled)
- A Quran Foundation developer account ([api-docs.quran.foundation](https://api-docs.quran.foundation))
- A Google Gemini API key

### 1. Clone the repository

```bash
git clone https://github.com/Noble-Knight-7/quran-connect.git
cd quran-connect
```

### 2. Install frontend dependencies

```bash
npm install
```

### 3. Configure environment variables

Create a `.env` file in the project root:

```env
REACT_APP_FIREBASE_API_KEY=your_key
REACT_APP_FIREBASE_AUTH_DOMAIN=your_domain
REACT_APP_FIREBASE_PROJECT_ID=your_project_id
REACT_APP_FIREBASE_STORAGE_BUCKET=your_bucket
REACT_APP_FIREBASE_MESSAGING_SENDER_ID=your_id
REACT_APP_FIREBASE_APP_ID=your_app_id
REACT_APP_API_BASE_URL=http://localhost:5000
```

### 4. Install and configure the backend

```bash
cd server
npm install
```

Create `server/.env`:

```env
GEMINI_API_KEY=your_gemini_key
QF_CLIENT_ID=your_qf_client_id
QF_CLIENT_SECRET=your_qf_client_secret
QF_REDIRECT_URI=http://localhost:3000/qf/callback
FIREBASE_SERVICE_ACCOUNT_JSON={"type":"service_account",...}
```

### 5. Run the app

In two terminals:

```bash
# Terminal 1 — backend
cd server && node index.js

# Terminal 2 — frontend
npm start
```

Open [http://localhost:3000](http://localhost:3000) in your browser.
